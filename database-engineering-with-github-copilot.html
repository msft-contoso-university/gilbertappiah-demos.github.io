GitHub Copilot Modes for Database Work

GitHub Copilot offers three interaction surfaces. Each mode unlocks different database capabilities depending on how you engage.

💬

Ask Mode (@mssql / @pgsql)

Chat with your database using the chat participant. Type @mssql or @pgsql followed by your question. Schema-aware suggestions, query generation, and explanations — all scoped to your connected database.

Schema Aware Conversational
🤖

Agent Mode

Copilot autonomously selects and executes MSSQL/PostgreSQL tools. Connect to databases, run queries, explore schemas, and modify data — all through natural language with user confirmation before each action.

Tool Execution Autonomous
✨

Inline Suggestions

Type T-SQL or PL/pgSQL in the editor and receive real-time completions. Highlight code to access the smart action icon for review, modification, or documentation. Use Ctrl+I for inline Copilot chat.

Auto-complete Smart Actions

Agent Mode vs. Ask Mode: In Ask Mode, use @mssql or @pgsql explicitly. In Agent Mode, Copilot auto-detects the MSSQL/PostgreSQL tools when those extensions are active — no prefix needed. Agent Mode also handles connections through natural language without requiring a pre-established profile.

MSSQL Extension — Copilot Features

The MSSQL extension ships with 11 Copilot-powered capabilities purpose-built for SQL Server, Azure SQL, and Fabric SQL.

🔍 Schema Explorer & Designer

Create, reverse-engineer, and evolve schemas. Generate CREATE TABLE, ORM models, migration scripts, and validation rules from natural language.

📝 Smart Query Builder

Build complex queries from plain English. Time-based analysis, multi-table joins, business insights, and ORM-aware generation across Prisma, EF Core, Sequelize, and SQLAlchemy.

⚡ Query Optimizer Assistant

Identify inefficient queries, interpret execution plans in plain English, suggest indexing strategies, restructure queries with CTEs, and optimize ORM queries.

💻 Code Generation

Generate T-SQL scripts, stored procedures, views, ORM migrations, and full-stack application code (Azure Functions, Node.js, Django, Next.js, Blazor).

💡 Business Logic Explainer

Explain stored procedures, views, UDFs, and ORM queries in natural language. Document business rules, debug procedural logic, and onboard new team members.

🛡 Security Analyzer

Detect SQL injection vulnerabilities, review access control, recommend data masking, and suggest encryption strategies. Works with T-SQL, Python, JavaScript, and ORMs.

🧪 Test Data Generator

Generate schema-aware INSERT statements or ORM seeders. Infer schemas from JSON, CSV, or TXT files. Create themed test data for development environments.

🌐 Localization Helper

Suggest collation settings, Unicode usage, and query patterns for multi-language and region-specific requirements.

💬 Chat & Inline Suggestions

Conversational @mssql chat for schema exploration, query writing, and relationship explanation. Plus real-time inline completions as you type T-SQL.

Supported Platforms: SQL Server 2019/2022/2025 (Windows, Linux, containers), Azure SQL Database, Azure SQL Managed Instance, SQL Server on Azure VMs, SQL database in Fabric, Fabric Data Warehouse, and Fabric Lakehouse SQL analytics endpoint.

Query Optimization with AI

From identifying slow queries to interpreting execution plans and recommending indexes, Copilot turns query tuning from a specialist skill into a conversation.

What Copilot Can Do

  • Rewrite inefficient queries with proper JOINs and index-friendly predicates
  • Suggest indexing strategies (composite, covering, filtered)
  • Interpret execution plans in plain English (both estimated and actual)
  • Restructure queries using CTEs for readability and performance
  • Optimize ORM queries across Prisma, EF Core, Sequelize, and SQLAlchemy
  • Detect N+1 query problems in code-first workflows
  • Compare execution metrics before and after optimization
Optimize Query
Optimize the following query: SELECT * FROM tasks WHERE project_id IN ( SELECT id FROM projects WHERE name LIKE '%Mobile%' ) AND assigned_user_id IN ( SELECT id FROM users WHERE role = 'engineer' ) ORDER BY created_at DESC;

Execution Plan Analysis

Both extensions provide execution plan visualization. Share the plan with Copilot for plain-English interpretation.

Copilot Prompt (MSSQL)
According to the execution plan, this query uses a nested loop join affecting performance. Explain why and suggest optimizations.
Copilot Prompt (PostgreSQL)
Run EXPLAIN ANALYZE on this query against the PostgreSQL taskify database and tell me what's slow.

Security Analysis

Copilot detects SQL injection vulnerabilities, reviews access control patterns, and recommends data protection strategies across T-SQL and application code.

SQL Injection Detection

-- Copilot identifies this as CRITICAL: SQL Injection vulnerability query = f"SELECT * FROM users WHERE id = {user_input}" -- Copilot recommends parameterized queries: -- SQL Server EXEC sp_executesql N'SELECT * FROM users WHERE id = @id', N'@id INT', @id = @user_id;
Security Review
Review the usp_GetUserData stored procedure for SQL injection vulnerabilities and suggest secure alternatives.

Additional Security Capabilities

  • Access Control Review: Evaluate role-based permissions, schema ownership, and procedure security (DEFINER vs INVOKER)
  • Data Protection: Recommend data masking, Always Encrypted, and TDE configurations
  • Authentication: Guide Entra ID integration, managed identity usage, and secure connection strings
  • Sensitive Data: Identify SELECT * on tables with PII and recommend column-specific access views
  • ORM Security: Detect unsafe patterns in Prisma, Sequelize, EF Core, and SQLAlchemy code
Copilot Prompt
What SQL Server options should I enable to protect customer data when using EF Core? Consider Always Encrypted, TDE, and connection encryption.
